## AWS 데이터 보기 [#view-aws-data]

설치 및 구성이 완료되면 AWS Service의 데이터에 뉴렐릭 플랫폼에서 직접 액세스할 수 있습니다. <DNT>**[one.newrelic.com &gt; All capabilities &gt; Amazon Web Services](https://one.newrelic.com/all-capabilities)**</DNT> 로 이동합니다.

* AWS 데이터 탐색에 대한 자세한 내용은 [인프라 통합에서 얻은 데이터를 이해하고 사용하는 방법](/docs/infrastructure/infrastructure-integrations/get-started/understand-use-data-infrastructure-integrations) 에 대한 문서를 확인하세요.
* 또한 [AWS 통합 지표에 대한 문서도 구체적으로 검토해](/docs/infrastructure/amazon-integrations/manage-aws-data/aws-integrations-metrics) 보시는 것이 좋습니다.

## Amazon CloudWatch 메트릭 스트림 및 API 폴링 [#api-polling]

지표 데이터를 보관하는 지표 스트림에 직접 [CloudWatch Metric Streams 연결되므로 통합을](/docs/infrastructure/amazon-integrations/connect/cloudwatch-metric-streams/aws-metric-stream-setup) 사용하는 것이 좋습니다.AWS 하지만 [API 폴링 통합을](/docs/infrastructure/amazon-integrations/connect/connect-aws-new-relic-infrastructure-monitoring/) 통한 계측이 필요한 스택이 있을 수도 있습니다. 예를 들어, 다음 AWS 서비스는 현재 CloudWatch Metric Streams 통합에서 지원되지 않습니다.

* AWS 클라우드 트레일
* AWS 상태
* AWS 신뢰할 수 있는 고문
* AWS 엑스레이
* [AWS VPC 흐름 로그](/docs/infrastructure/amazon-integrations/aws-integrations-list/aws-vpc-flow-logs-monitoring-integration/)
* [AWS 클라우드 포메이션](https://github.com/newrelic/cloudformation-partner-integration)

## 통합 및 AWS 비용 [#cloudwatch-billing]

이 섹션에서는 AWS 통합이 AWS 청구에 어떤 영향을 미치는지 설명하는 관련 AWS 문서로 이동합니다.

* [AWS CloudWatch 가격 책정 페이지에서는](https://docs.aws.amazon.com/AmazonCloudWatch/latest/monitoring/CloudWatch-Metric-Streams.html)메트릭 업데이트 수에 따라 AWS CloudWatch Metric Streams 가격을 정의합니다.
* 때때로 AWS 에서 뉴렐릭으로의 전달 방법으로 AWS Kinesis Data Firehose를 사용하므로 [AWS Firehose 가격 페이지를](https://aws.amazon.com/kinesis/data-firehose/pricing/) 검토하는 것이 좋습니다.
* AWS Config를 사용하여 리소스 관리자로 CloudWatch 지표를 강화하면 기록된 설정 항목 수에 따라 요금이 부과됩니다. 자세한 내용은 [AWS Config 가격 페이지를](https://aws.amazon.com/config/pricing/) 참조하세요.
* [AWS 리소스 그룹 태그 API](https://docs.aws.amazon.com/resourcegroupstagging/latest/APIReference/overview.html) 연관된 AWS 리소스에 정의된 사용자 정의 태그로 MintWorks를 강화합니다. 이는 폴링과 메트릭 스트림 모두에 영향을 미칩니다.

뉴렐릭은 대신 Polling 통합을 사용할 때 Amazon CloudWatch 사용하여 API 서비스에서 지표를 AWS API CloudWatch Metric Streams 가져옵니다. 일부 상황에서는 CloudWatch API에 대한 호출 수가 AWS에서 부여한 <DNT>**1 million free**</DNT> 한도를 초과하여 [CloudWatch 요금이 증가](/docs/infrastructure/amazon-integrations/troubleshooting/cloudwatch-billing-increase) 할 수 있습니다. 다음과 같은 경우 이런 일이 발생할 수 있습니다.

* API 폴링을 통해 추가 통합 활성화
* 해당 통합에 AWS 리소스 추가
* 더 많은 지역으로 통합을 확장합니다.

AWS 일부 서비스에 대해 향상된 모니터링을 제공하여 더 높은 빈도로 더 많은 지표를 수집할 수 있도록 합니다. 자세한 내용은 [RDS 향상된 모니터링 비용을](https://docs.aws.amazon.com/AmazonRDS/latest/UserGuide/USER_Monitoring.OS.overview.html#USER_Monitoring.OS.cost) 확인하세요.

## 지역 가용성 [#region-availability]

대부분의 AWS 서비스는 클라우드 리소스와 애플리케이션 간의 데이터 지연을 줄이기 위해 리전 엔드포인트를 제공합니다. New Relic은 중국을 제외한 모든 [AWS 리전](http://docs.aws.amazon.com/general/latest/gr/rande.html) 에 위치한 서비스 및 엔드포인트에서 모니터링 데이터를 얻을 수 있습니다.
